Программа для поиска дубликатов файлов с использованием блокового хэширования (CRC32 или MD5).

# Установка

Соберите проект и получите исполняемый файл `bayan`.  

Пример сборки через CMake:
```bash
mkdir build
cd build
cmake ..
cmake --build .
```
После сборки бинарник будет доступен в папке build.

# Использование

## Параметры командной строки
```
Опция	Описание
-h, --help	Показать справку
-d, --dir <dirs>	Директории для сканирования (может быть несколько)
-e, --exclude <dirs>	Директории для исключения (может быть несколько)
-r, --recursive <0/1>	Рекурсивный обход директорий (1 — рекурсивно, 0 — только указанная директория)
-s, --block-size <S>	Размер блока для чтения файлов в байтах (по умолчанию 4096)
-m, --mask <masks>	Маски файлов для сравнения (не зависит от регистра, может быть несколько)
--min-size <bytes>	Минимальный размер файлов для проверки (по умолчанию 0)
-a, --alg <0/1>	Алгоритм хэширования: 0 — CRC32, 1 — MD5
```

# Пример использования

## Рекурсивно искать дубликаты в директории /data с блоком 4096 байт, используя MD5
bayan -d /data -r 1 -s 4096 -a 1

## Исключить папку tmp и проверять только *.txt файлы
bayan -d /data -e /data/tmp -m "*.txt" -r 1

# Особенности работы:
    Файлы сравниваются блоками заданного размера S.
    Если размер файла не кратен S, блоки дополняются нулями.
    Блоки хэшируются выбранным алгоритмом (CRC32 или MD5).
    Чтение с диска производится лениво, минимальное количество данных.
    Результат: список групп дубликатов, каждая группа разделена пустой строкой.

# Пример вывода

Group:
  /data/file1.txt

Group:
